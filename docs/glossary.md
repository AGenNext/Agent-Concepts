# Agent Concepts Glossary

## Agent

A governed digital actor that can observe context, reason over intent, use approved capabilities, and produce accountable outcomes.

## Agent Concept

A stable semantic unit used to define how agents are named, bounded, governed, measured, and validated.

## Agent Identity

The persistent identity of an agent across systems, tenants, runtimes, and lifecycle states.

## Agent Role

The function an agent performs inside a system. Examples include tool, operator, provider, coordinator, auditor, and assistant.

## Agent Capability

A declared ability that an agent may use when permitted by policy and context.

## Agent Skill

A reusable capability implementation with a defined input, output, risk, and evidence requirement.

## Agent Runtime

The execution environment where an agent loop, tool call, workflow, or skill runs.

## Agent Kernel

The minimal stable core that manages execution semantics, state transition, isolation, and lifecycle.

## Agent Control

The layer that constrains agent action through objective, policy, approvals, budget, risk, and kill-switch rules.

## Agent Policy

A machine-evaluable rule that allows, denies, escalates, or constrains an agent action.

## Agent Governance

The accountability model that defines ownership, responsibility, auditability, trust, risk, and separation of duty.

## Agent Evidence

Recorded proof of agent input, reasoning boundary, decision, approval, action, result, and verification.

## Agent Trace

A time-ordered evidence record for agent execution.

## Agent Conformance

The validation process proving that an implementation follows the canonical concept, schema, policy, and runtime boundary.

## Agent Maturity

A staged measurement of how production-ready, governed, observable, and reliable an agent implementation is.

## Desired State

The declared target condition that a system should converge toward.

## Observed State

The measured real condition of a system at a point in time.

## Reconciliation

The process of comparing desired state and observed state, then safely reducing drift.

## Drift

A meaningful difference between declared intent and observed reality.

## Human Gate

A required human approval, review, or escalation before a risky or uncertain action proceeds.

## Kill Switch

A control that immediately suspends or restricts agent authority when safety, trust, policy, or conformance fails.
